# FT6336G 触摸屏 I2C 通信协议驱动说明文档

## 1. 概述

FT6336G 是一款单芯片自电容触摸屏控制器，支持单点触摸、手势识别或两点触摸。本说明文档旨在指导开发者基于 I2C 接口实现与 FT6336G 的通信，包括硬件连接、初始化流程、数据读取、中断处理及 FreeRTOS 任务同步机制。

**主要特性**：

- 支持 I2C 从机模式，时钟频率最高 400 kHz。
- 内置增强型 MCU，自动校准，抗干扰能力强。
- 三种工作模式：Active（主动扫描）、Monitor（监控）、Hibernation（休眠）。
- 通过 **INT** 引脚（中断输出，低电平有效）通知主机有触摸数据就绪。
- 通过 **RSTN** 引脚（复位输入，低电平有效）硬件复位芯片。
- 支持最多 **2 点** 触摸。

本驱动基于标准 I2C 读写操作，并结合外部中断和 FreeRTOS 任务通知实现高效数据采集。

## 2. 硬件连接

| FT6336G 引脚 | 功能               | 连接至主机                          |
| :----------- | :----------------- | :---------------------------------- |
| SDA          | I2C 数据线         | MCU I2C SDA 引脚                    |
| SCL          | I2C 时钟线         | MCU I2C SCL 引脚                    |
| INT          | 中断输出（低有效） | MCU 外部中断引脚 (下降沿触发)       |
| RSTN         | 复位输入（低有效） | MCU GPIO 输出引脚                   |
| VDDA         | 电源 2.8~3.6V      | 3.3V 电源                           |
| VDDD         | 数字电源 1.8~3.6V  | 3.3V 电源（或接 IOVCC）             |
| IOVCC        | I/O 电源 1.8~3.6V  | 3.3V 电源                           |
| VSS          | 地                 | GND                                 |
| VREF         | 内部参考电压       | 接 1μF 陶瓷电容到地（必须靠近芯片） |
| VDD5         | 电荷泵输出         | 接 1μF 陶瓷电容到地                 |

> **注意**：
>
> - **INT** 为开漏输出，**必须外接上拉电阻**（建议 4.7kΩ~10kΩ）至 IOVCC。
> - **RSTN** 低电平复位，低脉冲宽度应 **≥ 1ms**，复位后芯片进入 Active 模式。
> - **上电时序**：建议 VDDA 稳定后，再将 RSTN 由低拉高。

## 3. 初始化流程

### 3.1 硬件复位

```c
// 拉低 RSTN 至少 1ms，然后拉高
GPIO_WriteLow(RSTN_PIN);
vTaskDelay(pdMS_TO_TICKS(5));   // ≥1ms 即可
GPIO_WriteHigh(RSTN_PIN);
vTaskDelay(pdMS_TO_TICKS(50));  // 等待芯片内部初始化完成
```

### 3.2 I2C 接口初始化

配置 MCU 的 I2C 外设为主机模式，时钟频率 400 kHz（或 100 kHz），7 位设备地址为 **0x38**。

> **⚠️ 关于 I2C 地址的说明**：
>
> - FT6336G 的 **7 位设备地址** 是 `0x38`。
> - 若 MCU 库函数要求传入 **8 位地址**（包含读写位），则写操作为 `0x70`，读操作为 `0x71`。

### 3.3 读取芯片 ID（推荐）

通过读取寄存器 `0xA8` 获取厂商 ID，FT6336G 通常返回 `0x36`。

```c
uint8_t id = 0;
HAL_I2C_Mem_Read(&hi2c, 0x38, 0xA8, I2C_MEMADD_SIZE_8BIT, &id, 1, 100);
if (id == 0x36) {
    // 通信正常
} else {
    // 错误处理：检查硬件连接或尝试重新复位
}
```

### 3.4 配置中断模式（可选）

通过寄存器 `0xA4` 配置中断模式：

```c
// 设置为中断触发模式（默认），INT 在每个触摸事件触发一个脉冲
uint8_t g_mode = 0x01;
HAL_I2C_Mem_Write(&hi2c, 0x38, 0xA4, I2C_MEMADD_SIZE_8BIT, &g_mode, 1, 100);

// 设置为中断轮询模式，INT 在触摸期间持续为低
// uint8_t g_mode = 0x00;
```

## 4. I2C 通信协议

FT6336G 作为 I2C 从设备，支持标准的读写操作。

### 4.1 写寄存器

```c
// 向寄存器 addr 写入一个字节 data
HAL_I2C_Mem_Write(&hi2c, 0x38, addr, I2C_MEMADD_SIZE_8BIT, &data, 1, HAL_MAX_DELAY);
```

### 4.2 读寄存器

```c
// 从寄存器 addr 读取一个字节
HAL_I2C_Mem_Read(&hi2c, 0x38, addr, I2C_MEMADD_SIZE_8BIT, &data, 1, HAL_MAX_DELAY);
```

### 4.3 连续读取多个寄存器

```c
HAL_I2C_Mem_Read(&hi2c, 0x38, 0x02, I2C_MEMADD_SIZE_8BIT, buf, len, HAL_MAX_DELAY);
```

## 5. 寄存器映射（FT6336G）

以下寄存器映射表来源于 FocalTech FT6X36 系列数据手册第 26-28 页。

| 地址     | 名称           | 位描述                             | 说明                                                       | 访问 |
| :------- | :------------- | :--------------------------------- | :--------------------------------------------------------- | :--- |
| **0x00** | `DEV_MODE`     | bit[2:0]                           | 设备模式：`000`=Working，`100`=Factory                     | R/W  |
| **0x01** | `GEST_ID`      | bit[7:0]                           | 手势ID（见手势表）                                         | R    |
| **0x02** | `TD_STATUS`    | bit[3:0]                           | **触摸点数**（1~2有效）                                    | R    |
| **0x03** | `P1_XH`        | bit[7:6]=事件标志 bit[3:0]=X[11:8] | **事件标志**：`00`=按下，`01`=抬起，`10`=接触，`11`=无事件 | R    |
| **0x04** | `P1_XL`        | bit[7:0]=X[7:0]                    | X坐标低8位                                                 | R    |
| **0x05** | `P1_YH`        | bit[7:4]=Touch ID bit[3:0]=Y[11:8] | Touch ID（`0x0F`=无效）                                    | R    |
| **0x06** | `P1_YL`        | bit[7:0]=Y[7:0]                    | Y坐标低8位                                                 | R    |
| **0x07** | `P1_WEIGHT`    | bit[7:0]                           | 触摸压力值                                                 | R    |
| **0x08** | `P1_MISC`      | bit[7:4]=Touch Area                | 触摸面积                                                   | R    |
| **0x09** | `P2_XH`        | bit[7:6]=事件标志 bit[3:0]=X[11:8] | 第二点X高4位                                               | R    |
| **0x0A** | `P2_XL`        | bit[7:0]=X[7:0]                    | 第二点X低8位                                               | R    |
| **0x0B** | `P2_YH`        | bit[7:4]=Touch ID bit[3:0]=Y[11:8] | 第二点Y高4位                                               | R    |
| **0x0C** | `P2_YL`        | bit[7:0]=Y[7:0]                    | 第二点Y低8位                                               | R    |
| **0x0D** | `P2_WEIGHT`    | bit[7:0]                           | 第二点压力值                                               | R    |
| **0x0E** | `P2_MISC`      | bit[7:4]=Touch Area                | 第二点面积                                                 | R    |
| **0xA4** | `G_MODE`       | bit[7:0]                           | `0x00`=轮询模式，`0x01`=触发模式（默认）                   | R/W  |
| **0xA5** | `PWR_MODE`     | bit[7:0]                           | 当前电源模式                                               | R/W  |
| **0xA6** | `FIRMID`       | bit[7:0]                           | 固件版本                                                   | R    |
| **0xA8** | `FOCALTECH_ID` | bit[7:0]                           | 厂商ID（FT6336 = `0x36`）                                  | R    |
| **0xBC** | `STATE`        | bit[7:0]                           | 当前工作状态                                               | R/W  |

## 6. 手势识别

`GEST_ID` (0x01) 寄存器值：

| 值   | 手势                  |
| :--- | :-------------------- |
| 0x10 | 向上滑动 (Move Up)    |
| 0x14 | 向右滑动 (Move Right) |
| 0x18 | 向下滑动 (Move Down)  |
| 0x1C | 向左滑动 (Move Left)  |
| 0x48 | 放大 (Zoom In)        |
| 0x49 | 缩小 (Zoom Out)       |
| 0x00 | 无手势                |

## 7. 读取触摸位置数据

### 7.1 读取流程

1. 等待 INT 引脚变为低电平（或查询 `TD_STATUS` 寄存器）。
2. 一次性从 `0x02` 开始连续读取 6 个字节（单点）或 12 个字节（两点）。
3. 解析坐标和事件标志。

### 7.2 单点触摸读取示例

```c
uint8_t buf[6];
HAL_I2C_Mem_Read(&hi2c, 0x38, 0x02, I2C_MEMADD_SIZE_8BIT, buf, 6, 100);

uint8_t touch_num = buf[0] & 0x0F;          // TD_STATUS 低4位
uint8_t event_flag = (buf[2] >> 6) & 0x03;  // P1_XH 高2位

if (touch_num > 0 && event_flag != 0x01) {  // 非抬起事件
    // X坐标 = (P1_XH[3:0] << 8) | P1_XL
    uint16_t x = ((buf[2] & 0x0F) << 8) | buf[3];
    // Y坐标 = (P1_YH[3:0] << 8) | P1_YL
    uint16_t y = ((buf[4] & 0x0F) << 8) | buf[5];
    uint8_t touch_id = (buf[4] >> 4) & 0x0F;  // Touch ID
    
    // 上报坐标 (0~4095)
}
```

### 7.3 两点触摸读取示例

```c
uint8_t buf[12];
HAL_I2C_Mem_Read(&hi2c, 0x38, 0x02, I2C_MEMADD_SIZE_8BIT, buf, 12, 100);

uint8_t touch_num = buf[0] & 0x0F;

// 第一点 (寄存器 0x03~0x06)
if (touch_num >= 1) {
    uint8_t event1 = (buf[2] >> 6) & 0x03;
    uint16_t x1 = ((buf[2] & 0x0F) << 8) | buf[3];
    uint16_t y1 = ((buf[4] & 0x0F) << 8) | buf[5];
}

// 第二点 (寄存器 0x09~0x0C)
if (touch_num >= 2) {
    uint8_t event2 = (buf[8] >> 6) & 0x03;   // buf[8] = P2_XH
    uint16_t x2 = ((buf[8] & 0x0F) << 8) | buf[9];
    uint16_t y2 = ((buf[10] & 0x0F) << 8) | buf[11];
}
```

> **注意**：此款 2.8 寸屏幕模组通常仅支持单点触摸（取决于模组厂商配置），但 FT6336G 芯片本身支持两点。

## 8. 中断处理与 FreeRTOS 任务通知

### 8.1 INT 引脚配置

- 将 INT 引脚配置为**下降沿触发**的外部中断。
- 建议中断优先级较低（如 5），以免阻塞系统调度。

### 8.2 中断服务函数（ISR）

```c
void EXTI_INT_IRQHandler(void) {
    if (__HAL_GPIO_EXTI_GET_IT(INT_PIN) != RESET) {
        __HAL_GPIO_EXTI_CLEAR_IT(INT_PIN);
        BaseType_t xHigherPriorityTaskWoken = pdFALSE;
        vTaskNotifyGiveFromISR(touchTaskHandle, &xHigherPriorityTaskWoken);
        portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
    }
}
```

### 8.3 触摸处理任务

```c
void touch_task(void *pvParameters) {
    while (1) {
        if (ulTaskNotifyTake(pdTRUE, portMAX_DELAY) == 1) {
            ft6336g_read_touch();
        }
    }
}
```

## 9. 完整示例代码

```c
// ========== 宏定义 ==========
#define FT6336G_I2C_ADDR        0x38

// ========== 寄存器地址 ==========
#define FT_REG_DEV_MODE         0x00
#define FT_REG_GEST_ID          0x01
#define FT_REG_TD_STATUS        0x02
#define FT_REG_P1_XH            0x03
#define FT_REG_P1_XL            0x04
#define FT_REG_P1_YH            0x05
#define FT_REG_P1_YL            0x06
#define FT_REG_P1_WEIGHT        0x07
#define FT_REG_P1_MISC          0x08
#define FT_REG_P2_XH            0x09
#define FT_REG_P2_XL            0x0A
#define FT_REG_P2_YH            0x0B
#define FT_REG_P2_YL            0x0C
#define FT_REG_P2_WEIGHT        0x0D
#define FT_REG_P2_MISC          0x0E
#define FT_REG_G_MODE           0xA4
#define FT_REG_PWR_MODE         0xA5
#define FT_REG_FIRMID           0xA6
#define FT_REG_FOCALTECH_ID     0xA8
#define FT_REG_STATE            0xBC

// ========== I2C 读写函数 ==========
uint8_t ft6336g_read_reg(uint8_t reg) {
    uint8_t data = 0;
    HAL_I2C_Mem_Read(&hi2c1, FT6336G_I2C_ADDR, reg, I2C_MEMADD_SIZE_8BIT, &data, 1, 100);
    return data;
}

uint8_t ft6336g_read_regs(uint8_t reg, uint8_t *buf, uint8_t len) {
    return HAL_I2C_Mem_Read(&hi2c1, FT6336G_I2C_ADDR, reg, I2C_MEMADD_SIZE_8BIT, buf, len, 100);
}

void ft6336g_write_reg(uint8_t reg, uint8_t data) {
    HAL_I2C_Mem_Write(&hi2c1, FT6336G_I2C_ADDR, reg, I2C_MEMADD_SIZE_8BIT, &data, 1, 100);
}

// ========== 初始化函数 ==========
void ft6336g_init(void) {
    // 1. 硬件复位
    HAL_GPIO_WritePin(RSTN_GPIO_Port, RSTN_Pin, GPIO_PIN_RESET);
    HAL_Delay(5);   // ≥1ms
    HAL_GPIO_WritePin(RSTN_GPIO_Port, RSTN_Pin, GPIO_PIN_SET);
    HAL_Delay(50);

    // 2. 读取芯片 ID 校验通信
    uint8_t id = ft6336g_read_reg(FT_REG_FOCALTECH_ID);
    if (id != 0x36) {
        // 错误处理
        while (1) { /* 挂起或报错 */ }
    }

    // 3. 配置中断模式（可选）
    // ft6336g_write_reg(FT_REG_G_MODE, 0x01);  // 中断触发模式（默认）

    // 4. 配置外部中断
    HAL_NVIC_SetPriority(EXTI_INT_IRQn, 5, 0);
    HAL_NVIC_EnableIRQ(EXTI_INT_IRQn);
}

// ========== 读取触摸数据 ==========
void ft6336g_read_touch(void) {
    uint8_t buf[6];
    if (ft6336g_read_regs(FT_REG_TD_STATUS, buf, 6) != HAL_OK) {
        return;
    }

    uint8_t touch_num = buf[0] & 0x0F;
    uint8_t event_flag = (buf[2] >> 6) & 0x03;

    if (touch_num > 0 && event_flag != 0x01) {
        uint16_t x = ((buf[2] & 0x0F) << 8) | buf[3];
        uint16_t y = ((buf[4] & 0x0F) << 8) | buf[5];
        // 上报坐标
    } else {
        // 无触摸或抬起
    }
}

// ========== 中断服务函数 ==========
void EXTI_INT_IRQHandler(void) {
    if (__HAL_GPIO_EXTI_GET_IT(INT_Pin) != RESET) {
        __HAL_GPIO_EXTI_CLEAR_IT(INT_Pin);
        BaseType_t xHigherPriorityTaskWoken = pdFALSE;
        vTaskNotifyGiveFromISR(touchTaskHandle, &xHigherPriorityTaskWoken);
        portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
    }
}

// ========== FreeRTOS 任务 ==========
void touch_task(void *pvParameters) {
    while (1) {
        if (ulTaskNotifyTake(pdTRUE, portMAX_DELAY) == 1) {
            ft6336g_read_touch();
        }
    }
}
```

## 10. 常见问题排查

| 现象           | 可能原因与解决措施                                           |
| :------------- | :----------------------------------------------------------- |
| I2C 无法通信   | 检查 I2C 地址（7位/8位）、上拉电阻、电源电压、RST 时序（≥1ms） |
| INT 不触发     | 检查中断配置（下降沿触发）、上拉电阻、触摸是否正常           |
| 坐标异常或跳变 | 检查电源纹波、排线连接，尝试重新初始化                       |
| 触摸无响应     | 可能处于 Hibernation 模式，通过硬件复位唤醒                  |
