你的 **2.8寸 SPI 接口** ILI9341 屏幕，默认工作在 **65K 色（16位 RGB 5-6-5）** 模式，数据通过 **4线 SPI** 总线传输。

------

## 📖 1. 接口模式确认（来源：手册第 26 页，Table 7-1）

你的屏幕标称“SPI 串口”，对应的硬件引脚配置为 `IM[3:0]`。

对于 **4线 8位数据 SPI 接口**，硬件配置为 `IM3=0, IM2=1, IM1=1, IM0=0`（或 IM3=1, IM2=1, IM1=1, IM0=0）。

根据手册，使用 4线 SPI 时，数据传输格式如下：

| IM3  | IM2  | IM1  | IM0  | MCU接口模式                | 使用的引脚               |
| :--- | :--- | :--- | :--- | :------------------------- | :----------------------- |
| 0    | 1    | 1    | 0    | **4线 8位数据串行接口 I**  | SCL, SDA, D/CX, CSX      |
| 1    | 1    | 1    | 0    | **4线 8位数据串行接口 II** | SCL, SDI, D/CX, SDO, CSX |

> 你的模组通常使用第一种（共用 SDA 读写）。

------

## 🎨 2. 像素颜色格式（来源：手册第 63-64 页，第 7.6.2 节）

在 4线 SPI 模式下，ILI9341 支持 **2种** 颜色深度，通过命令 **3Ah（Pixel Format Set）** 的 `DBI[2:0]` 位来设置。

### 模式 A：65K 色（16位/像素 RGB 5-6-5）—— 这是绝大多数 SPI 屏幕的默认模式

寄存器 `3Ah` 设置 `DBI[2:0] = 101`

**数据包格式（每 1 个像素占 2 个字节）：**

| 传输顺序    | D/CX位   | 字节内容 (MSB先发)        | 对应颜色位             |
| :---------- | :------- | :------------------------ | :--------------------- |
| **第1字节** | 1 (数据) | `R4 R3 R2 R1 R0 G5 G4 G3` | Red高5位 + Green高3位  |
| **第2字节** | 1 (数据) | `G2 G1 G0 B4 B3 B2 B1 B0` | Green低3位 + Blue低5位 |

**组成结果：**

- Red: 5 bits (0~31)
- Green: 6 bits (0~63)
- Blue: 5 bits (0~31)
- 总计：16 bits = 65536 种颜色

### 模式 B：262K 色（18位/像素 RGB 6-6-6）

寄存器 `3Ah` 设置 `DBI[2:0] = 110`

**数据包格式（每 1 个像素占 3 个字节）：**

| 传输顺序    | D/CX位   | 字节内容 (MSB先发)  | 对应颜色位 |
| :---------- | :------- | :------------------ | :--------- |
| **第1字节** | 1 (数据) | `R5 R4 R3 R2 R1 R0` | 全部Red    |
| **第2字节** | 1 (数据) | `G5 G4 G3 G2 G1 G0` | 全部Green  |
| **第3字节** | 1 (数据) | `B5 B4 B3 B2 B1 B0` | 全部Blue   |

- 总计：18 bits = 262144 种颜色
- **缺点**：传输速度慢（多传 1/3 数据），一般只在 RGB 并行接口下使用，SPI 下很少用。

## 🔄 3. 颜色顺序（RGB 还是 BGR？来源：手册第 127 页，命令 `36h`）

这是一个**大坑**！很多屏幕颜色不对（红蓝反了），就是因为这个寄存器。

命令 **36h（Memory Access Control）** 的第 3 位（Bit 3）叫做 **BGR**。

- `BGR = 0`：颜色顺序为 **RGB**（正常）
- `BGR = 1`：颜色顺序为 **BGR**（红蓝交换）

**绝大多数国产 2.8寸 SPI 模组，出厂默认 BGR = 1！**
所以你传输纯红色 `0xF800` 时，屏幕显示纯蓝色。需要根据你购买的模组厂家提供的初始化代码来确认这一位。

------

## 📐 4. 屏幕坐标与分辨率（来源：手册第 110-113 页）

- **分辨率**：240（列） x 320（行） 像素。
- **设置列地址**：命令 `2Ah`（Column Address Set）。
  - 起始列（SC）：默认 `0x0000`
  - 结束列（EC）：默认 `0x00EF`（即 239）
- **设置行地址**：命令 `2Bh`（Page Address Set）。
  - 起始行（SP）：默认 `0x0000`
  - 结束行（EP）：默认 `0x013F`（即 319）

写入像素数据时，列地址和行地址会自动递增。

## 🚀 5. 软件上如何正确初始化并设置颜色格式

如果你想在代码中**明确指定**颜色格式，请按以下流程（这是通用的 ILI9341 SPI 初始化序列）：

c

```
// 1. 硬件复位（拉低再拉高，等待 10ms）
// 2. 退出睡眠模式 (Sleep Out)
write_command(0x11);
delay_ms(120);  // 必须等待 120ms

// 3. 设置颜色格式 (Pixel Format Set)
write_command(0x3A);
write_data(0x05);  // 0x05 = 16位/像素 RGB565 (推荐)
// write_data(0x06); // 0x06 = 18位/像素 RGB666 (SPI下不推荐)

// 4. 设置内存访问控制 (Memory Access Control)
write_command(0x36);
write_data(0x08);  // 0x08 通常表示 BGR=1 (即 BGR 顺序)，如果你的红色显示为蓝色，请检查这一位！
// 常见值：0x00 (RGB) 或 0x08 (BGR)

// 5. 开启显示 (Display ON)
write_command(0x29);
```
